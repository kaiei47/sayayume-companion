> 永愛学園 AI恋愛シミュレーション

## コンセプト

**「AIとの共同恋愛小説」×「恋愛シミュレーションゲーム」**

2つのモードを自由に行き来し、パーソナルメモリで体験が繋がる。

---

## 2つのモード

### チャットモード（日常）
- いつでもさや/ゆめとリアルタイム会話
- 時間帯で口調・テンションが変化（朝は眠い、夜は素直）
- ストーリーモードの体験が「共有の思い出」としてメモリに入り、会話が深まる
- 「今の彼女」とのリアルな関係

### ストーリーモード（AI自由展開型シナリオ）
- 好きなストーリーを選んでプレイ
- 各ストーリーにミッション（達成条件）
- **選択肢ボタンは使わない。ユーザーが自分の言葉で行動する**
- **ストーリーの大きな流れはAIに設定するが、展開はユーザーの言動で無限に分岐**
- **ミッション達成はAIが会話内容から自動判定**（正解ルートは固定ではない）
- ミッション完遂のために何度もリプレイ → 毎回違うアプローチが試せる
- 体験がパーソナルメモリに保存 → チャットモードに反映

### なぜ選択肢ボタンを使わないのか
- AIの最大の強み＝「何を言ってもいい自由度」
- 選択肢は従来ギャルゲーの制約。AIでやる意味がない
- 「自分の言葉でキャラの心を動かす」体験こそがAI恋愛シミュの本質
- 同じミッションでも攻略法が無限 → リプレイ性が桁違い
- 例: ミッション「手を繋ぐ」
  - ストレートに「手繋いでいい？」→ 成功
  - 「寒いね...」と言って自然に手を近づける → 成功
  - 「迷子にならないように」と冗談で → 成功
  - 全部違うルートだけど全部正解

### 親密度の減衰（超重要）

**6時間ごとに親密度が少しずつ下がる。**

| Lv帯 | 6時間ごとの減衰 | 1日放置で |
|------|----------------|----------|
| Lv1-3 | -2pt | -8pt |
| Lv4-6 | -5pt | -20pt |
| Lv7-8 | -8pt | -32pt |
| Lv9-10 | -12pt | -48pt |

**設計意図：**
- 高レベルほど減衰が大きい = 「恋人関係を維持するには毎日の努力が必要」
- 1日サボるとLv9→Lv8に落ちる可能性 → 毎日ログインの強い動機
- ただしLv1以下には絶対に落ちない（完全リセットはしない）

### 親密度が下がる行動

| 行動 | 減少量 | キャラの反応 |
|------|--------|------------|
| 傷つけること言う | -10〜-30pt | 悲しむ/怒る/黙る |
| 約束を破る（ストーリー内） | -15pt | 「待ってたのに...」 |
| 6時間放置 | -2〜-12pt | 次回「最近会えなくて寂しかった」 |
| 24時間放置 | 上記×4 | 「忘れちゃった...？」 |
| 他のキャラとデートバレ | -20pt | 嫉妬イベント発生 |

### 親密度が上がる行動

| 行動 | 増加量 |
|------|--------|

| ストーリーモード完了 | +10〜30pt |

### ストーリーモードの実装方式

**ストーリー = 「シチュエーション設定 + ミッション定義 + 特別システムプロンプト」**

実際のセリフはAIが毎回生成する。固定テキストではない。

```typescript
interface Story {
  id: string;
  title: string;
  character: 'saya' | 'yume' | 'duo';
  playTime: number; // 分
  difficulty: 1 | 2 | 3 | 4 | 5;
  requiredIntimacy: number; // 必要Lv

  // AIへのシチュエーション注入
  setting: string; // 「井の頭公園。秋の午後。紅葉がきれい」
  systemPromptAddition: string; // ストーリー固有の指示

  // ミッション定義
  missions: {
    id: string;
    description: string; // 「手を繋ぐ」
    hint: string; // 「自然な流れで手を差し伸べてみよう」
    detectPrompt: string; // AIに「このミッションは達成された？」と判定させる
    reward: { intimacy: number; image?: string };
  }[];

  // 全ミッション完遂報酬
  completionReward: {
    intimacy: number;
    specialImage: string; // 特別CGプロンプト
    title?: string; // 称号
  };
}
```

### ミッション達成判定

**AIに判定させる方式（バックエンド変更最小）：**

ストーリーモード中、各ユーザーメッセージ送信後にGeminiに「このミッションは達成されたか？」をサイドで判定させる。

```
判定プロンプト例：
「以下の会話履歴を読み、ミッション『手を繋ぐ』が達成されたか判定してください。
ユーザーが自然な流れでキャラクターと手を繋ぐ行動をとり、
キャラクターがそれを受け入れた場合に達成とします。
回答は YES or NO のみ。」
```

### 親密度によるリアクション変化（超重要・ゲームの核心）

**同じ行動でも、親密度によってキャラの反応が劇的に変わる。**
これが「リアルな恋愛」のドキドキを再現する核心設計。

AIのシステムプロンプトに親密度レベルを注入し、それに応じたリアルな反応を生成させる。

#### 例：ミッション「手を繋ぐ」

**Lv1-2（知らない人〜顔見知り）:**
- ユーザーが手を繋ごうとする → キャラは拒否・引く・怒る
- しつこくする → 「もう帰る」ストーリー強制終了。親密度-100
- **まだ早い。まず友達になることから。**

**Lv3-4（クラスメイト〜友達）:**
- ユーザーが手を繋ごうとする → 驚く・戸惑う・「え、なんで急に...」
- 上手い言い方をすれば受け入れる場合も（AIが文脈で判断）
- 成功率30%くらい。失敗しても親密度-5程度

**Lv5-6（仲良し〜特別な人）:**
- 自然な流れで手を繋げる。「寒いね」→「...うん」で成立
- 成功率70%。キャラも内心嬉しい
- 親密度+10〜15

**Lv7-8（両想い〜恋人）:**
- ほぼ確実に成功。キャラから手を繋いでくることも
- 「ねぇ、手」（さやが自分から）
- 親密度+5

**Lv9-10（運命の人〜永遠）:**
- デフォルトで手を繋いでる。離すと「え、なんで離すの？」
- もはやミッションではない。日常

#### ストーリーモードでの親密度チェック

**ストーリー選択時の親密度 = そのストーリー内でのキャラの反応基準。**

同じ「井の頭公園デート」でも：
- Lv2で挑戦 → 友達として行く。距離感あり。ミッション達成困難
- Lv6で挑戦 → いい雰囲気。自然にミッション達成可能
- Lv9で挑戦 → 完全にカップルデート。全ミッション余裕+追加イベント発生

**→ 同じストーリーでも親密度が違えば全く別の体験になる = リプレイ性が無限**

#### AIプロンプトへの親密度注入

```
ストーリーモード開始時にシステムプロンプトに追加：

「現在の親密度: Lv{X}（{レベル名}）
Lv{X}のあなたは{ユーザー名}に対して以下のように振る舞います：
- 距離感: {近い/普通/遠い}
- スキンシップ: {自分から/{条件付きでOK/基本NG/絶対NG}
- 本音: {話す/少し話す/隠す}
- 呼び方: {あだ名/名前呼び/苗字+さん}

重要ルール:
- 親密度に見合わない急な距離の詰め方をされたら、リアルに拒否・戸惑い・怒りを表現すること
- 特にLv1-3でのスキンシップ要求は明確に拒否し、しつこい場合はストーリーを終了させること
- これは「リアルな恋愛」を再現するための核心設計」
```

#### ストーリー強制終了の設計

キャラを怒らせた/傷つけた場合、ストーリーが途中で終わる可能性がある。

```
終了トリガー:
- 親密度に見合わないスキンシップを3回連続で試みる
- 暴言・侮辱的な発言
- キャラが「帰る」「もう話したくない」と宣言

終了時:
- 親密度-50〜-100
- ストーリー未完了（ミッション未達成扱い）
- チャットモードで「この前は...ごめん。ちょっと怖かった」（翌日以降）
- 関係修復にはチャットモードで誠実な会話が必要
```

```

  ↓
```
