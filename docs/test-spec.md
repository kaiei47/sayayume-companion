### TC-001: 無効プランでの画像生成防止 [P0 SECURITY]

**対象ファイル**: `src/app/api/chat/route.ts` L453

**問題**:
`PLAN_LIMITS[userPlan]?.imageGeneration !== false` の評価ロジックが脆弱。
DBから予期しないプラン値（例: 空文字、null、'enterprise'）が返った場合、
`PLAN_LIMITS[x]` = undefined → `undefined?.imageGeneration` = undefined → `undefined !== false` = **true**
→ フリープランユーザーが画像生成できてしまう。

**テスト手順**:
1. DBの `subscriptions.plan` を手動で `'enterprise'`（未定義プラン）に設定
2. フリープランユーザーとしてチャットで写真リクエストを送る
3. 画像が生成されないこと、アップグレードメッセージが表示されることを確認

**期待結果**: `!== false` → `=== true` に修正後、未定義プランでは画像生成しない

**ステータス**: ❌ NG → 修正対象

### TC-002: 会話メッセージ数の正確な追跡 [P1]

**対象ファイル**: `src/app/api/chat/route.ts` L551

**問題**:
`message_count: history.length + 2` は context compression 後に破綻する。
- 圧縮なし: history = 全メッセージ（user message含む）→ `history.length + 2` = 1件多い
- 圧縮あり: history = 最新8件のみ → `message_count` が常に約10に固定される
→ 会話履歴UIでのメッセージ数表示が狂う。

**テスト手順**:
1. さやとのチャットで20回以上メッセージを送る（コンテキスト圧縮を発動させる）
2. 管理画面 `/admin` でメッセージ数を確認
3. `conversations` テーブルの `message_count` を直接確認

**期待結果**: 実際のメッセージ数と `message_count` が一致する

**ステータス**: ❌ NG → 修正対象

### TC-003: 新規ユーザーのStripeサブスクボーナストークン付与 [P1]

**対象ファイル**: `src/app/api/stripe/webhook/route.ts` L79

**問題**:
```typescript
const { data: balance } = await getSupabaseAdmin()
  .from('token_balances')
  .select('balance, total_purchased')
  .eq('user_id', userId)
  .single();
if (balance) { ... // ボーナストークン付与
```
`token_balances` レコードがない新規ユーザーの場合、`balance` = null → `if (balance)` が false →
**ボーナストークンが付与されない**。課金しても特典が機能しない重大バグ。

**テスト手順**:
1. テスト環境で新規アカウントを作成（token_balancesレコードなし）
2. Basicプランのチェックアウトを完了（テストカード 4242...）
3. `token_balances` テーブルを確認

**期待結果**: 新規ユーザーでもボーナストークン（Basic: 200、Premium: 500）が付与される

**ステータス**: ❌ NG → 修正対象

### TC-005: デュオチャット メッセージID衝突 [P2]

**対象ファイル**: `src/app/chat/duo/page.tsx` L300-310

**問題**:
さやとゆめのメッセージに `Date.now()` と `Date.now() + 1` を ID として使用。
高速な連続レスポンス時に React reconciliation でレンダリングが崩れる可能性。

**テスト手順**:
1. デュオチャットで連続して複数メッセージを素早く送る
2. メッセージ表示が正しい順序・人物で表示されるか確認
3. Reactの key prop 重複警告がコンソールに出ないことを確認

**期待結果**: コンソール警告なし、メッセージ表示順序が正常

**ステータス**: ❌ NG → 修正対象
